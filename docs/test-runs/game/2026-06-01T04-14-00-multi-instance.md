# Test run — multi-instance (3 servers · 1 game · 6 players · 10 rounds)

**Date:** 2026-06-01
**Build:** `claude/multiple-server-instances-C0L36` (Redis-backed shared state)
**Topology:** 3 uvicorn instances (A:8101, B:8102, C:8103) sharing one Redis;
`TELEMETRY_ENABLED=0`. Six real browsers (Playwright MCP, Chromium) — two per
instance.

| # | Player | Browser instance | Server |
|---|--------|------------------|--------|
| 1 | Alpha (host) | `playwright` | A:8101 |
| 2 | Beta | `playwright-guest` | B:8102 |
| 3 | Gamma | `playwright-g2-host` | C:8103 |
| 4 | Delta | `playwright-g2-guest` | A:8101 |
| 5 | Epsilon | `playwright-g3-host` | B:8102 |
| 6 | Zeta | `playwright-g3-guest` | C:8103 |

## Results — all PASS

| Check | Result |
|-------|--------|
| Host creates game on A; code `DDXRG` | ✅ |
| 5 players join via deep-link across B and C (cross-instance) | ✅ deep-link pre-filled the code on every instance |
| Host lobby shows all 6 players (fan-out A←B,C) | ✅ |
| A player on instance C also sees all 6 in the lobby | ✅ |
| Host starts; all 6 transition to the game board | ✅ |
| 6 players roll concurrently across 3 instances for 10+ rounds | ✅ ~16s/round, no hangs, rounds auto-advanced |
| Target cycles 1→2→3→4→5→6→1 with round number | ✅ round 12 → target 6 |
| Single winner per round (no double-wins under simultaneous rolls) | ✅ total wins == completed rounds |
| All 6 browsers agree on final win tally | ✅ identical on every instance |
| Live cross-instance reconnect (reload Zeta on C, token auto-reconnect) | ✅ back in game, round/players intact |
| Server exceptions across A/B/C | ✅ zero |
| Browser console errors (host + a guest) | ✅ zero |

**Final tally** (identical on all 6 clients): Alpha 1, Beta 3, Gamma 1,
Delta 4, Epsilon 0, Zeta 2 — sum **11** across 11 completed rounds (round 11
finished during the stop gap). The win count exactly matching the completed-round
count is the key evidence the Lua win compare-and-set (`gamestore.try_finish_round`)
prevents double-winners even with six rollers racing across three processes.

## Notes / limitations
- Chrome channel unavailable (Google's apt/download path blocked); used
  Playwright's bundled Chromium symlinked to the `chrome` path the MCP expects.
- Rolling was automated by injecting a paced (350 ms) roll-button clicker per
  browser; the client's own reveal/`roll_done` handshake was exercised normally.
- Screenshots: `.playwright-mcp/multi-01-lobby-6players.png`,
