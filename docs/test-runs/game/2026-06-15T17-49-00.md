## Notes

- **First run with auth steps (24–29).** The full WebAuthn ceremony (CDP Virtual Authenticator → `navigator.credentials.create()` → server `py_webauthn` verify → JWT → onboarding) works end-to-end. Sign-in with existing credentials, sign-out, and both negative checks (wrong username 404, duplicate registration 409) all passed.
- **RP ID must be `localhost` for local dev.** The default `WEBAUTHN_RP_ID=tensies.app` in `docker-compose.yml` causes `navigator.credentials.create()` to fail with "The relying party ID is not a registrable domain suffix of, nor equal to the current domain." The first sign-up attempt hit this; restarting with `WEBAUTHN_RP_ID=localhost` fixed it. This is expected (WebAuthn spec requirement) — only affects local testing. The prod compose uses `${WEBAUTHN_RP_ID:?set WEBAUTHN_RP_ID}` which must be set to the deploy domain.
- **Throwaway-tab session clobbering** hit again in Step 16 (Beta's localStorage cleared by earlier edge-case tabs). Re-seeded with captured `BETA_PID`/`BETA_TOKEN`/`GAME_CODE` — recovered cleanly. Prior log suggested using g2-guest for edge-case checks.
- **pg_data/prom_data volume conflict** on stack switch (dev→prod). Reset with `docker volume rm` as in prior runs.
- **`.env.prod` needed temp auth vars** (`JWT_SECRET`, `WEBAUTHN_RP_ID`) for prod build Steps 30–34. Added then removed after test. The prod compose file's `${JWT_SECRET:?set JWT_SECRET}` correctly blocks startup when missing.
