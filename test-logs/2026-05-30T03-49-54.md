Scope: implemented + verified the brainstormed pause/real-world scenarios, **excluding
telemetry**. Harness: `uvicorn` on :8888 (no Docker) + local Postgres; bundled Chromium
via isolated `browser.newContext()` contexts (chrome-channel MCP unavailable).

## Product fixes shipped this round

- **#1 Host abandonment during a pause** — `drop_player`, while paused, no longer just
  early-returns: if the *host* is the one gone past `DISCONNECT_GRACE`, it transfers the
  host role (and resume control) to a still-connected player (old host keeps their seat).
  A paused game can no longer be frozen until the 1 h cap by an absent host.
- **#2 Round won during the `ROUND_WIN_DELAY` window** — `delayed_broadcast` now defers
  the round advance if a pause landed (`round_advance_pending`); `handle_pause` calls the
  new `advance_round()` helper on resume. Pause truly freezes a just-won round.
- **#3 Pause landing mid-reveal on a roller** — client holds the newer snapshot in
  `state.postRevealState`; `tryReveal` re-routes via `showFor()` after the reveal so a
  non-host roller lands on the wait screen instead of being stranded on the board.

## Notes

- **#12 is flaky in-batch, solid in isolation.** It failed with an 8–15 s `waitForFunction`
  timeout when run right after other tests, but passed **3/3 standalone** and once more in
  the full edge run after isolating. Root cause is Chromium context-creation contention
  (one-off slow page load), not a 3-player product bug. The throwaway runner bumps the
  screen-wait timeout and closes each test's contexts in a `finally` to reduce cascades.
- **#15 product behavior verified deterministically.** The in-node `uvicorn` restart was
  unreliable in this sandbox (wrong CWD → `Could not import main` / missing `static/`, and
  recurring local-Postgres drops). The restart *does* bring the server back; to verify the
  client path without that flakiness, the test points the guest's saved code at a missing
  game and drops the socket — the reconnect loop then gets "Game not found" → `expireSession`
  → landing ("Connection failed", `localStorage` cleared). Identical to the post-restart path.
- **#4 (re-pause resets the 1 h cap)** — kept as intended; an actively-toggling host is
  present, not abandoned. Documented in CLAUDE.md.
- **#13 (host transfer → new host can pause)** — the transfer is covered by unit test #1;
  the new host seeing the host-only toggle is the same `renderMenu(snap.host === myId)` path
  already exercised. Not separately browser-tested (real transfer needs a 60 s grace).
- **Not addressed (by design / out of scope):** #16 mobile background (needs a real device),
  #17 many-paused-games load (loadtest territory), #22 menu auto-open on a network blip and
  #23 countdown drift (cosmetic, acceptable). Telemetry untested per request.
